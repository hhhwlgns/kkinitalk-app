## 문서 목적

이 문서는 끼니톡 전면 개편의 지속적인 기준 문서다. 작업이 길어지거나 대화가 요약되더라도 이 파일을 먼저 읽어 제품 의도, 결정사항, 남은 질문과 진행 상태를 복구한다.

- 구현 중 새로 확정한 내용은 `확정된 제품 방향`에 반영한다.
- 임시 판단이나 확인이 필요한 내용은 `미확정 사항`에 기록한다.
- 실제 파일 변경과 검증 결과는 `작업 로그`에 날짜순으로 기록한다.
- 기존 요구를 변경할 때는 이전 문장을 조용히 덮어쓰지 말고 `변경 이력`에도 이유를 남긴다.
- 현재는 요구사항만 기록한 상태이며, 이 문서 생성 외의 구현 작업은 시작하지 않았다.

## 제품 개편의 핵심

이번 작업은 단순한 화면 리디자인이 아니라 역할별 정보 구조와 핵심 사용 흐름을 전면 재설계하는 작업이다.

- 어르신 모드: 오늘 먹을 것, 먹은 것, 복약할 것과 AI 도움을 최대한 단순하게 제공한다.
- 보호자 모드: 어르신 상태를 더 자세히 파악하고 관리하며 필요한 식단을 보낼 수 있게 한다.
- 어르신은 복잡한 메뉴 탐색 없이 홈에서 오늘 필요한 정보와 행동을 대부분 해결해야 한다.
- 식사와 복약은 날짜를 중심으로 탐색한다.
- 촬영은 홈과 식사 화면 안에서 제공하고, 하단 중앙 핵심 버튼은 AI 음성 대화로 변경한다.
- 보호자는 어르신보다 더 상세한 수치, 추세, 상태와 기록을 확인한다.
- 보호자 모드에는 이커머스와 어르신 관리 기능을 신설한다.

## 확정된 제품 방향

### 1. 어르신 하단 내비게이션

하단 내비게이션은 3개로 간소화한다.

1. 왼쪽: 식사
2. 가운데: AI 음성 대화 플로팅 버튼
3. 오른쪽: 복약

기존 중앙 사진 촬영 버튼은 AI 보이스챗 버튼으로 대체한다. 식사 사진 촬영은 홈의 오늘 식사 카드와 식사 화면의 끼니별 영역에서 실행한다.

`내 정보`는 하단 내비게이션에서 제거한다. 홈 상단의 이름이나 프로필 영역 등 별도 진입점을 제공한다.

### 2. 어르신 홈

어르신 홈은 오늘 하루를 관리하는 통합 대시보드다.

#### 상단

- 어르신 이름과 따뜻한 인사
- 오늘 날짜
- 내 정보 진입점
- 보호자가 보낸 도시락 또는 선물이 있으면 최상단 배너로 표시

도시락 배너에는 보낸 보호자, 메뉴, 도착 예정 시간과 배송 상태를 보여준다. 자세히 보기를 통해 영양정보와 메뉴 상세로 이동할 수 있다.

#### 하루 목표 영양소

오늘 섭취량과 하루 목표량을 즉시 파악할 수 있게 시각화한다.

- 열량
- 탄수화물
- 단백질
- 지방
- 나트륨

숫자만 나열하지 않고 원형 진행률, 막대, 상태 색상과 `충분해요`, `조금 부족해요`, `주의가 필요해요` 같은 쉬운 문구를 사용한다. 나트륨은 적게 섭취했다고 무조건 부족으로 판정하지 않으며 영양소별 판정 기준을 따로 둔다.

#### 다음 메뉴 추천

건강 프로필, 오늘 섭취량과 최근 영양 추이를 반영해 대화형 문장으로 다음 메뉴를 제안한다.

예시:

> 단백질이 조금 부족해요. 다음 끼니에는 두부구이가 어떠세요?

추천에는 추천 이유와 음식 사진을 보여줄 수 있으며, 식사 상세 또는 이커머스 상품으로 연결할 수 있다.

#### 영양소 추이

최근 영양소 변화를 그래프 또는 직관적인 시각화로 홈에서 바로 확인한다. 기본 범위는 최근 7일을 우선 검토한다. 홈에는 중요한 추세를 간단히 보여주고 식사 화면에서 상세 정보를 제공한다.

#### 오늘 복약 카드

- 아침약, 점심약, 저녁약의 완료 상태
- 시간대별 전체 약 개수와 복용한 약 개수
- 다음 복용 시간
- 남아 있는 약
- 복약 상세 화면 진입

시간대마다 약이 하나라고 가정하지 않는다. 각 시간대에 여러 약이 있을 수 있다.

#### 오늘 식사

- 아침, 점심, 저녁의 촬영 사진 표시
- 기록이 없으면 사진 촬영 진입점 표시
- 사진을 누르면 해당 식사의 상세 분석으로 이동
- 상세 분석에는 음식 목록, 영양소, 적합도, 주의사항과 다음 메뉴 추천 표시

### 3. 어르신 식사 화면

선택한 날짜의 식사와 영양 상태를 통합해서 보여준다.

#### 날짜 탐색

- 왼쪽 위: 전날
- 오른쪽 위: 달력 또는 다음날
- 달력에서 날짜 직접 선택
- 과거 날짜로 이동하면 다음날 이동 수단 제공

달력과 다음날 버튼을 동시에 어떻게 제공할지는 UI 설계 단계에서 확정한다. 날짜 제목을 눌러 달력을 여는 방식도 후보로 둔다.

#### 도시락/선물 배너

선택한 날짜에 보호자가 보낸 식단이 있으면 화면 상단에 표시한다.

#### 탭 1: 목표 영양소 현황

- 해당 날짜의 하루 목표
- 현재 섭취량
- 남은 양
- 적정, 부족, 과다 상태
- 영양소별 시각화

#### 탭 2: 영양 분석과 다음 메뉴 추천

부족하거나 과한 영양소를 쉬운 문장으로 설명하고 다음 메뉴를 추천한다.

예시:

> 단백질이 부족해요. 다음 메뉴로 제육볶음은 어떠세요?

추천은 질환, 복용약, 약-음식 상성, 알레르기, 기피 음식, 연하 곤란, 오늘과 최근 식사 기록을 함께 고려해야 한다.

#### 탭 3: 끼니별 기록

- 아침, 점심, 저녁 기록
- 끼니별 음식 사진과 음식 목록
- 영양 상태 및 주의사항
- 미기록 끼니의 촬영 버튼
- 사진 선택 시 식사 상세 분석 이동

한 끼에 여러 음식과 필요 시 여러 사진을 표현할 수 있는 구조를 고려한다.

### 4. 어르신 AI 음성 대화

하단 중앙의 주요 버튼이다. 초기에는 실제 AI가 연결되지 않더라도 보이스봇 UI와 목업 대화 흐름을 구현할 수 있다.

예상 요청:

- 오늘 무엇을 먹으면 좋은지 질문
- 오늘 복약 여부 확인
- 몸 상태 말하기 및 체크인
- 식사 기록 시작
- 보호자에게 알리기
- 오늘 영양 상태 질문

### 5. 어르신 복약 화면

#### 상단 주의사항

현재 복용약과 음식의 상성 또는 복용 주의사항을 가장 먼저 보여준다.

예시:

> 혈압약을 드시는 동안 자몽과 자몽주스는 피해주세요.

일반 안내, 오늘 식사와 관련된 주의, 즉시 확인이 필요한 경고를 시각적으로 구분한다. 모든 의료·복약 문구에는 참고용 성격과 의사·약사 상담 안내를 유지한다.

#### 시간대 구분

- 아침약
- 점심약
- 저녁약
- 필요시 복용약은 확장 후보

각 시간대에는 여러 약이 포함될 수 있다. 사용자는 약별로 선택하거나 모두 복용 완료 처리할 수 있어야 한다.

약별로 다음 상태를 고려한다.

- 복용 완료
- 미복용
- 나중에 알림
- 건너뜀
- 복용 시간 수정

장기적으로 약 이름, 용량, 복용 개수, 복용 시간, 식전/식후, 주의 음식과 활성 상태를 저장할 수 있어야 한다.

### 6. 어르신 내 정보

#### 최상단 연결 코드

- 보호자 연결용 어르신 코드
- 크게 보기
- 공유하기
- 새 코드 만들기

#### 연결된 보호자 목록

한 어르신에게 여러 보호자가 연결될 수 있다.

- 보호자 이름
- 관계
- 연결 상태
- 공유 범위
- 고위험 정보 공유 여부
- 연결 해제

#### 건강 프로필

- 이름, 나이, 성별
- 질환
- 복용약
- 알레르기
- 연하 곤란
- 기피 음식
- 체중
- 식욕
- 고위험 정보 공유 동의

### 7. 보호자 하단 내비게이션

보호자 알림은 하단 탭에서 제거하고 화면 우측 상단의 종 아이콘으로 이동한다. 읽지 않은 알림 개수 배지를 제공한다.

새 하단 내비게이션의 기본안:

1. 홈
2. 기록
3. 복약
4. 이커머스
5. 어르신 관리

기존 리포트 탭은 제거하고 `어르신 관리`로 대체한다. 주간 리포트 기능은 삭제하지 않고 홈, 기록 또는 어르신 관리 내부로 이동한다.

### 8. 보호자 홈

- 현재 보고 있는 어르신과 대상 전환
- 우측 상단 알림
- 긴급하거나 확인이 필요한 상태
- 어르신의 오늘 목표 영양소 시각화
- 오늘 식사 사진과 상세 분석 진입
- 오늘 복약 상태
- 최근 영양, 식사, 복약과 컨디션 추세

어르신 홈과 시각적 언어는 공유하지만 더 자세한 수치와 상태를 제공한다.

예시:

> 오늘 나트륨은 목표의 128%입니다. 점심 순두부찌개가 전체 나트륨의 54%를 차지했습니다.

### 9. 보호자 기록

어르신 식사 화면과 유사한 날짜 중심 구조를 사용하되 다음 상세정보를 추가한다.

- 일간, 주간, 월간 보기
- 목표 대비 초과·부족량
- 최근 평균 비교
- 식사별 영양소 기여도
- 기록 누락
- 반복적으로 부족하거나 과한 영양소
- 질환 관련 위험
- 보호자 메모와 확인 요청
- 주간 리포트

### 10. 보호자 복약

어르신 복약 화면과 동일한 시간대 구조를 사용하되 관리 기능을 강화한다.

- 약별 예정 시간과 실제 복용 시간
- 지연과 미복용 상태
- 최근 복약 이행률
- 연속 미복용
- 식사와의 상성
- 약 추가 및 수정
- 일정 변경
- 주의 음식 등록
- 기록 정정
- 어르신에게 확인 요청

어르신이 직접 남긴 복용 기록과 보호자가 대신 기록하거나 수정한 내역을 구분하고 수정 주체와 시간을 남겨야 한다.

### 11. 보호자 이커머스

어르신의 건강 상태와 영양 부족을 바탕으로 보호자가 맞춤 식단이나 도시락을 골라 배송하는 목업 기능을 신설한다.

#### 이커머스 홈

- 오늘의 추천 식단
- 어르신 맞춤 추천
- 부족 영양소 보완 메뉴
- 질환별 식단
- 인기 메뉴
- 정기배송
- 최근 주문

카테고리 후보:

- 저염식
- 당뇨 관리식
- 고단백 식단
- 부드러운 식단
- 일반 건강식
- 죽 또는 연하 보조식
- 반찬 세트

#### 상품 데이터

- 상품명과 이미지
- 가격
- 배송 가능일
- 열량, 단백질, 나트륨 등 영양정보
- 메뉴 구성
- 알레르기 정보
- 추천 이유
- 적합한 건강 조건
- 복용약 및 음식 주의사항

#### 주문 흐름

상품 선택 → 어르신 선택 → 배송 날짜 선택 → 배송지 확인 → 선물 메시지 → 주문 완료 → 어르신 홈과 식사 화면에 배너 표시

실제 결제와 배송 API는 현재 범위에 포함하지 않고 AsyncStorage 기반 목업데이터와 주문 상태로 구현한다.

### 12. 어르신 관리

기존 리포트 탭을 대체하는 보호자용 관리 화면이다.

- 어르신 기본정보
- 연결 상태와 최근 활동
- 건강 프로필 조회
- 질환, 알레르기, 기피 음식, 체중과 식욕 등 상태 수정
- 복약 일정 관리
- 보호자 메모와 상태 확인 기록
- 주간 건강 요약 및 기존 리포트 기능

민감한 건강정보는 수정한 사람과 시간을 기록하는 구조를 고려한다. 어르신과 보호자의 수정 권한을 구분한다.

### 13. 보호자 알림

알림은 우측 상단 종 아이콘에서 접근하고 관련 상세 화면으로 바로 이동한다.

알림 유형:

- 식사 누락
- 영양 위험
- 복약 누락 또는 지연
- 약-음식 충돌
- 컨디션 악화
- 어르신 도움 요청
- 배송 상태와 도시락 수령
- 건강 프로필 변경

## 경쟁 앱 캡처 분석 및 참고 원칙

> 확인일: 2026-07-19  
> 확인 자료: `reference/inout` 8장, `reference/pasta` 4장, `reference/pillyze` 11장

캡처는 기능과 정보 구조를 연구하기 위한 참고 자료다. 특정 앱의 색상, 캐릭터, 문구, 화면 배치나 그래픽 표현을 그대로 복제하지 않는다. 끼니톡의 어르신 접근성, 보호자 연결, 복약과 의료 안전이라는 제품 목적에 맞게 재해석한다.

### 인아웃에서 참고할 점

인아웃 캡처에서 확인한 주요 화면은 개인 목표 생성, 일간 식단 현황, 영양소 상세, 끼니별 기록, 기간별 그래프와 월간 달력이다.

#### 가져올 패턴

- 상단에서 날짜를 중심으로 이전날과 다음날을 빠르게 탐색하는 구조
- 섭취 열량과 목표 열량을 `현재 / 목표` 형식으로 즉시 비교하는 방식
- 탄수화물, 단백질, 지방을 같은 색 체계로 반복 표시해 학습 부담을 줄이는 방식
- 아침, 점심, 저녁, 간식을 큰 카드로 나누고 기록 여부를 즉시 보여주는 방식
- 일간, 주간, 월간을 전환해 같은 지표를 다른 기간으로 확인하는 구조
- 달력 날짜마다 기록 여부나 수치를 작은 상태로 표시하고 선택 날짜로 이동하는 흐름
- 기록이 없는 끼니는 큰 `+`로 다음 행동을 명확하게 제시하는 방식

#### 끼니톡 적용

- 어르신 식사 화면의 날짜 탐색, 끼니 카드와 영양소 현황의 기본 골격에 활용한다.
- 보호자 기록 화면의 일간·주간·월간 전환과 달력 요약에 활용한다.
- 열량만 크게 강조하지 않고 단백질과 나트륨 등 어르신 건강에 중요한 지표를 동등하게 다룬다.
- 어르신에게 `탄·단·지` 축약어만 제시하지 않고 전체 명칭을 함께 사용한다.

#### 그대로 가져오지 않을 점

- 작은 글씨와 낮은 대비의 보조정보
- 한 화면에 많은 서비스 메뉴와 프로모션을 배치하는 구성
- 체중 감량과 칼로리 소모를 중심으로 한 문구
- 어르신이 이해하기 어려운 축약어 및 복잡한 통계
- 기록 보상이나 광고가 핵심 건강 행동보다 앞서는 구조

### 파스타에서 참고할 점

파스타 캡처에서 확인한 주요 화면은 식사 사진 분석, 분석 결과 저장, 예상 혈당 그래프, AI 해설과 후속 질문, AI 챗봇이다.

#### 가져올 패턴

- 음식 사진을 화면의 중심에 크게 보여주고 분석 결과와 직접 연결하는 방식
- 분석 직후 핵심 수치, 한 줄 평가, 실행 가능한 조언을 한 카드에 묶는 구조
- 그래프 위에 `예상 최고`, `일정 시간 후` 같은 핵심 수치를 별도 카드로 먼저 설명하는 방식
- 분석 결과를 단순 판정으로 끝내지 않고 다음 끼니에 할 행동으로 연결하는 문장 구조
- 사용자가 이어서 물어볼 수 있는 추천 질문 칩
- AI 대화에서 사용자 질문과 답변을 명확히 구분하고 음성 입력 진입점을 제공하는 방식

#### 끼니톡 적용

- 식사 사진 상세 화면은 `사진 → 핵심 판정 → 주요 영양소 → 이유 → 다음 행동` 순서로 구성한다.
- 어르신에게는 긴 설명보다 한두 문장의 핵심 조언을 먼저 보여주고, 보호자에게는 근거와 상세 그래프를 펼쳐서 제공한다.
- AI 음성 대화에는 `오늘 뭐 먹으면 좋아?`, `아침약 먹었어?` 같은 추천 질문을 큰 버튼으로 제공한다.
- 다음 메뉴 추천은 `무엇이 문제인지`와 `그래서 무엇을 먹을지`가 한 흐름으로 이어져야 한다.

#### 그대로 가져오지 않을 점

- 실제 측정값으로 오인할 수 있는 정밀한 예측 그래프. 끼니톡의 mock 또는 추정 데이터는 추정임을 명확히 표시한다.
- 어르신에게 한 화면 가득 제공되는 긴 AI 설명
- 연령대나 특정 사용자군과 비교해 의학적 신뢰를 암시하는 표현
- 실제 근거가 없는 혈당 예측 및 위험 단정
- 전문적인 어두운 대시보드 스타일을 어르신 화면에 적용하는 것

### 필라이즈에서 참고할 점

필라이즈 캡처에서 확인한 주요 화면은 개인화 온보딩, 관리 카드 편집, 일일 식단 카드, 맞춤 탄단지 목표, 식사 점수, AI 코치와 대화형 분석이다.

#### 가져올 패턴

- 온보딩 입력값을 바탕으로 개인 목표를 계산하는 과정을 단계적으로 보여주는 방식
- 하루 섭취 목표를 단일 숫자가 아니라 적정 범위로 표시하는 방식
- 아침, 점심, 저녁 사진을 하나의 식단 카드에서 바로 확인하는 구성
- 결과에 `적정`, `주의` 같은 상태 라벨을 붙이고 색으로 보조하는 방식
- AI 코치가 수치 요약 뒤에 쉬운 언어로 핵심 문제와 다음 행동을 설명하는 방식
- 개인화 근거를 `왜 이 목표인지` 설명하는 정보 카드
- 관리 카드를 사용 목적에 맞게 선택하거나 순서를 조정하는 발상

#### 끼니톡 적용

- 어르신 홈의 목표 영양소, 아침·점심·저녁 사진 카드와 다음 행동 안내에 활용한다.
- 목표량은 가능한 경우 하나의 절대값보다 안전한 적정 범위를 시각화한다.
- 보호자 홈에는 요약 점수보다 실제 상태와 근거를 우선하고, 점수를 사용하더라도 상세 근거로 이동할 수 있게 한다.
- AI 답변은 어르신의 이름, 건강 프로필과 오늘 기록을 반영하되 진단처럼 표현하지 않는다.
- 어르신 홈 카드 편집은 초기 범위에서 제외하되, 보호자용 상세 대시보드 개인화의 확장 후보로 둔다.

#### 그대로 가져오지 않을 점

- 어르신에게 여러 건강 기능을 작은 카드로 빽빽하게 제공하는 구성
- 식단을 하나의 점수로 지나치게 단순화하는 방식
- 체중 감량과 단식을 중심으로 한 목표 체계
- 마스코트와 장식이 핵심 정보보다 강하게 보이는 구성
- 건강 데이터를 근거로 확정적인 결과를 말하는 AI 문구

### 끼니톡에 조합할 최종 참고 방향

세 앱에서 각각 다음 강점을 취한다.

```text
인아웃
└─ 날짜 탐색 + 목표 대비 현황 + 끼니별 기록 + 기간별 추이

파스타
└─ 큰 식사 사진 + 분석 근거 + 행동 가능한 다음 조언 + AI 후속 질문

필라이즈
└─ 개인화 목표 + 적정 범위 + 한눈에 보는 식단 카드 + AI 코치 설명

끼니톡 고유 가치
└─ 어르신 접근성 + 복약 관리 + 약-음식 주의 + 보호자 연결·관리 + 도시락 선물
```

따라서 끼니톡의 어르신 식사 경험은 다음 순서로 설계한다.

1. 날짜를 선택한다.
2. 목표 영양소와 현재 상태를 한눈에 본다.
3. 아침·점심·저녁 사진과 기록 여부를 확인한다.
4. 부족하거나 주의할 영양소를 쉬운 문장으로 이해한다.
5. 다음 메뉴 추천 또는 사진 촬영이라는 명확한 다음 행동을 선택한다.
6. 필요하면 AI 음성 대화로 질문한다.

보호자 경험은 같은 데이터를 더 자세히 확장한다.

1. 오늘 확인이 필요한 위험과 누락을 먼저 본다.
2. 목표 대비 수치와 최근 추이를 확인한다.
3. 끼니 사진과 영양 분석의 근거를 확인한다.
4. 복약 상태 및 약-음식 주의를 함께 확인한다.
5. 어르신에게 확인 요청을 보내거나 맞춤 도시락을 선택한다.

### 경쟁 앱 분석으로 추가된 설계 원칙

- 홈 요약과 상세 화면은 동일한 색과 상태 표현을 사용한다.
- 모든 그래프에는 그래프를 읽지 않아도 이해할 수 있는 한 줄 요약을 함께 제공한다.
- 어르신 화면에서 그래프는 한 번에 한 가지 질문에 답해야 한다.
- `0`과 `기록 없음`을 구분한다. 먹지 않은 것과 기록하지 않은 것은 같은 상태가 아니다.
- 목표는 가능하면 적정 범위로 표현하고 초과와 부족의 의미를 영양소별로 다르게 처리한다.
- AI 분석은 판정에서 끝나지 않고 다음 행동으로 연결한다.
- 식사 사진은 단순 첨부물이 아니라 기록 탐색과 상세 분석의 주 진입점이다.
- 보호자에게는 더 많은 정보를 보여주되 핵심 위험 → 근거 → 상세 순서의 정보 위계를 지킨다.
- 경쟁 앱의 다이어트 문법을 그대로 사용하지 않고 어르신의 균형 섭취, 안전한 복약과 지속적인 돌봄을 중심으로 문구를 작성한다.

## 안전 및 UX 원칙

- 어르신 화면은 큰 글씨, 높은 대비와 큰 터치 영역을 유지한다.
- 사용자 문구는 따뜻하고 정중한 한국어로 작성한다.
- 색상, 간격과 글꼴은 디자인 토큰을 사용한다.
- 모든 의료·영양·약 상성 안내는 참고용이며 진단과 처방을 대체하지 않는다.
- 약 변경 또는 중단을 유도하지 않고 의사·약사 상담을 안내한다.
- 긴급 상황의 119 안내를 유지한다.
- 고위험 정보 공유는 어르신 동의를 전제로 한다.
- 어르신과 보호자의 데이터 수정 권한을 혼동하지 않는다.

## 미확정 사항

1. 어르신 홈과 `식사` 탭의 정보가 중복될 때 각각 어느 수준까지 보여줄지.
2. 과거 날짜에서 오른쪽 상단을 `다음날`로 바꿀 경우 달력 진입점을 어디에 유지할지.
3. 어르신 내 정보의 정확한 진입 방식: 프로필 사진, 이름 영역 또는 상단 별도 버튼.
4. AI 음성 대화의 1차 구현 범위: UI 목업만 구현할지, 음성 입력·출력까지 연결할지.
5. 도시락 배송지가 기본적으로 어르신 집인지, 보호자 집도 선택 가능해야 하는지. 현재 문맥상 기본 배송지는 어르신 집으로 해석한다.
6. 보호자 하단 내비게이션을 최종적으로 5개 모두 유지할지.
7. 주간 리포트를 홈, 기록, 어르신 관리 중 어디에 주 진입점으로 둘지.
8. 보호자가 수정할 수 있는 건강 상태와 어르신 동의가 필요한 항목의 정확한 권한 범위.
9. 보호자 여러 명 및 어르신 여러 명 연결을 이번 개편에서 실제 지원할지, UI와 데이터 구조만 대비할지.
10. 이커머스의 범위를 상품 탐색·주문 목업까지로 할지, 장바구니·결제 목업까지 포함할지.

- 보호자 기록에 공통 날짜 탐색을 적용하고 일간 상세와 최근 7일 추이를 전환할 수 있게 했다.
- 일간 기록에서 목표 영양소, 끼니 사진, 복약 이행과 컨디션을 같은 날짜 기준으로 확인한다.
- 보호자 복약 화면에 오늘 복약 이행 수치, 음식 주의 약 개수와 상단 알림 진입점을 추가했다.
- 보호자가 약을 수정할 때 기존 용량·시간대·주의사항 선택 필드가 사라지지 않도록 기존 레코드를 병합한다.
- 보호자 핵심 화면 변경 후 `npx tsc --noEmit`, `smoke:guardian`, `smoke:history`, `smoke:medication`, `smoke:redesign`이 통과했다.
- 전체 회귀 검증으로 `smoke:db`, `onboarding`, `checkin`, `meal`, `suggestion`, `medication`, `history`, `profile`, `guardian`, `consent`, `redesign`을 순차 실행했고 모두 통과했다.
- Expo Go에서 번들 완료 후 홈으로 돌아가는 현상을 진단했다. Android 번들 생성과 Expo 의존성 검사는 정상이어서 앱 시작 전 MSW 초기화를 기본 비활성화했다.
- MSW는 `EXPO_PUBLIC_USE_MOCKS=true`일 때만 켜고, mock DB 부트스트랩 실패 시 빈 화면 대신 원인을 표시하는 초기화 오류 화면을 추가했다.
- 초기화 안전성 변경 후 `npx tsc --noEmit`이 통과했다.

## 변경 이력

### 2026-07-19 — 실제 화면 검수 후 2차 고도화 분석

- 웹 실제 화면에서 나트륨 관련 문구와 단일 나트륨 추이가 반복되어 제품이 종합 영양 관리가 아니라 저염 관리 앱처럼 보이는 문제를 확인했다.
- 홈은 카드 수가 많고 각 카드의 시각적 무게가 비슷해 `오늘 상태`, `지금 할 일`, `세부 기록`의 우선순위가 한눈에 구분되지 않는다.
- 영양 상태는 열량·탄수화물·단백질·지방·식이섬유·나트륨을 함께 보되, 모든 수치를 같은 크기로 나열하지 않고 종합 상태 → 핵심 변화 1~2개 → 전체 상세 순서로 단계화한다.
- 나트륨은 부족 영양소처럼 보충을 유도하지 않고 상한 관리 지표로 다룬다. 추천 문구는 단백질, 채소·식이섬유, 에너지, 탄수화물·지방 균형과 수분·나트륨 주의를 종합해 매일 달라져야 한다.
- 인아웃에서 `하루 상태를 하나의 큰 시각물로 압축`, `끼니 사진을 즉시 기록하는 구조`, `일·주·월 기간 전환`을 참고한다. 다이어트용 체중·칼로리 잔여 문법은 그대로 사용하지 않는다.
- 파스타에서 `사진을 크게 확인 → 한 줄 판정 → 판정 근거 → 다음 행동`, `그래프 위에 결론 수치`, `AI 추천 질문으로 후속 탐색` 구조를 참고한다.
- 필라이즈에서 `개인화된 적정 범위`, `탄단지 비율의 하나짜리 조합 시각화`, `카드별 관리 모듈`, `AI 코치의 기억과 후속 대화` 구조를 참고한다. 점수와 경고를 과도하게 사용해 불안을 키우는 방식은 피한다.
- 어르신 홈의 첫 화면은 `오늘 균형 한눈에` 히어로, AI가 고른 오늘의 핵심 행동, 다음 복약, 아침·점심·저녁 사진 순으로 재구성한다. 상세 영양 표와 7일 추이는 펼치기 또는 식사 상세로 이동한다.
- 종합 영양 시각화는 5~6개의 개별 게이지를 첫 화면에 늘어놓는 대신 `균형 접시` 또는 하나의 조합형 시각화와 쉬운 한 줄 결론을 우선 검토한다. 세부 화면에서는 적정 범위 막대와 기간별 추이를 제공한다.
- AI 화면은 안내 카드형 목업에서 대화방형 인터페이스로 전면 교체한다. 사용자·AI 말풍선, 대화 이력, 추천 질문, 텍스트 입력, 전송, 마이크 입력 상태, 음성 답변 재생, 식사 사진 첨부를 한 흐름에 둔다.
- AI 답변은 `공감/확인 → 기록을 반영한 짧은 답 → 구체적인 다음 행동 → 필요한 후속 질문` 순서를 사용하고, 긴 의료 설명은 접어서 제공한다.
- 보호자 홈은 단순히 어르신 화면에 수치를 더하는 방식에서 벗어나 `지금 확인할 것 → 오늘 종합 상태 → 식사·복약 근거 → 7일 변화 → 연락/식단 보내기` 순서로 재설계한다.
- 보호자 시각화에는 영양 균형, 끼니 규칙성, 복약 이행, 컨디션을 분리해서 보여주고, 위험·미기록·정상 상태를 명확히 구분한다. 동일한 경고를 홈·기록·복약에 반복하지 않는다.
- 다음 구현은 개별 카드 장식보다 정보 구조와 시각화 체계를 먼저 확정한 뒤 `공통 영양 요약 → 어르신 홈 → AI 대화 → 보호자 홈 → 상세 화면` 순으로 진행한다.
